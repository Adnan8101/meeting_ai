Overview

AI Meeting Agent is an intelligent automation platform that leverages advanced AI to analyze meeting transcripts, extract actionable insights, and automatically integrate with your favorite productivity tools.

The platform uses Google's Gemini AI to process natural language and identify key decisions, action items, and summaries from your meeting discussions. It then seamlessly distributes this information across your team's workflow tools.

Key Features

AI-Powered Analysis

Automatically extract summaries, decisions, and action items from meeting transcripts using Google Gemini AI.

Team Collaboration

Create teams, invite members, and automatically distribute meeting summaries via email.

Multi-Platform Integration

Connect with Trello, Slack, and Jira to automate task creation and notifications.

Secure & Private

Enterprise-grade security with encrypted credentials and secure OAuth integrations.

Architecture

System Components

  • •
    Frontend Layer: Server-side rendered templates using Jinja2 with TailwindCSS for styling
  • •
    Application Layer: Flask web framework handling routing, authentication, and business logic
  • •
    Database Layer: SQLAlchemy ORM with SQLite (dev) or PostgreSQL (production)
  • •
    AI Layer: Google Gemini 2.0 for natural language processing
  • •
    Integration Layer: REST API clients for Trello, Slack, and Jira

Database Schema

User → username, email, password_hash, team_id
Team → name, slack_webhook_url, members[]
TrelloCredentials → token, trello_username, user_id
JiraCredentials → jira_url, email, api_token, user_id
TrelloCard → card_id, task_description, assignee, due_date

Data Flow

Meeting Analysis Workflow

  1. 1
    Transcript Input: User pastes meeting transcript into the web interface
  2. 2
    Automation Selection: User enables desired integrations (Trello, Slack, Jira, Email)
  3. 3
    AI Processing: Gemini AI analyzes transcript and extracts structured data (summary, decisions, action items)
  4. 4
    Results Display: Analysis results are rendered on the web page in a structured format
  5. 5
    Automation Execution: Background tasks create Trello cards, send emails, post to Slack, and create Jira issues based on selected options

API Routes

GET /

Home page - Meeting transcript analysis interface

POST /analyze

Analyzes transcript and executes selected automations

GET /team

Team management page - View/create team and invite members

GET /integrations

Integration management - Connect/disconnect Trello, Slack, and Jira

GET /get_lists/<board_id>

Fetch Trello lists for a specific board (JSON response)

GET /get_jira_projects

Fetch all Jira projects for authenticated user (JSON response)

AUTH /login, /register, /logout

Authentication routes for user management

Integration Details

Trello

Automatically creates cards for each action item with assignee and due date information.

  • • Uses OAuth 1.0a for secure authentication
  • • Stores user tokens encrypted in database
  • • Allows board and list selection per analysis

Slack

Posts formatted meeting summaries to a designated Slack channel using Incoming Webhooks.

  • • Uses Incoming Webhooks (no OAuth required)
  • • Rich message formatting with Slack Block Kit
  • • Team-level integration (not user-level)

Jira

Creates issues for action items in specified Jira projects with custom issue types.

  • • Uses Jira Cloud REST API with API tokens
  • • Dynamic project and issue type selection
  • • Supports multiple Jira instances per user
